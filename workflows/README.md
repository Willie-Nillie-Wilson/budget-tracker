# workflows/

Markdown SOPs — the **Workflows** layer of WAT. Plain-language instructions, the way you'd brief a teammate.

Each workflow (`some_task.md`) should define:
- **Objective** — what this accomplishes
- **Inputs** — what's needed to start
- **Tools** — which scripts in `tools/` to run, and in what order
- **Outputs** — what gets produced and where it goes (usually a cloud service)
- **Edge cases** — known failures, rate limits, quirks, how to handle them

Keep these current. Update a workflow whenever you learn something new — don't overwrite without asking.
